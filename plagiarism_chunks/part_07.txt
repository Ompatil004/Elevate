It ensures that the transition from system design to software development is structured and systematic. 5.4.1 Project Planning Strategy The Elevate project follows the Agile Scrum software development methodology. This iterative approach is necessary to accommodate the testing and tuning required for the machine learning algorithms. The project was developed using an iterative Agile Scrum approach, where features were implemented and tested in multiple development cycles. The initial cycles focus on establishing backend database schemas and processing the Kaggle datasets required for training the XGBoost models. Subsequent cycles introduce the integration of the React frontend, followed by the real-time pose tracking implementation using the Google MediaPipe engine. This parallel development strategy allows the backend machine learning layer to be tested independently before being bridged to the consumer-facing user interface. 5.4.2 Module Planning To ensure codebase maintainability and separation of concerns, the project is divided into four distinct modules: 1. Authentication and Profile Module: Responsible for handling user registration, credential storage, and JSON Web Token (JWT) session generation. This module securely maps user biometrics to their database profiles. 2. Machine Learning Inference Module: The artificial intelligence layer. This module operates on a Python FastAPI server and loads pre-trained XGBoost regression models to generate customized workout and nutritional matrices based on user inputs. 3. Real-Time Vision Module: The biomechanical tracking engine. It captures live webcam video streams and processes them through the MediaPipe framework to extract 3D spatial joint coordinates. This module executes locally to prevent video data from being transmitted across the network, ensuring enhanced user privacy. 4. User Interface and Dashboard Module: The consumer-facing React application. It visually renders the data generated by the backend engines, providing workout plans, meal breakdowns, and form correction overlays during live exercise sessions. 5.4.3 Resource Planning The successful execution of the Elevate platform requires a balanced selection of software and hardware resources. Software Resources: Frontend Framework: React.js powered by Vite. Backend Gateways: Node.js and Express.js for routing and authentication. AI Processing Server: Python 3.10+ utilizing FastAPI. Machine Learning Libraries: XGBoost for regression, Scikit-Learn for preprocessing, and Pandas for dataframe manipulation. Computer Vision: Google MediaPipe for kinematic tracking. Database Management: MongoDB Atlas for NoSQL document storage. Hardware Resources: Development Workstations: A computer system with sufficient processing power and memory to support application development, model training, and testing. Client Hardware: A standard web camera and a modern web browser to execute the client-side computer vision models. 5.4.4 Risk Assessment and Mitigation Identifying potential architectural and operational risks early ensures the system remains stable. 1.
