When Google MediaPipe detects a human body within the webcam feed, it outputs 33 distinct 3D landmarks (represented as X, Y, Z coordinates). To translate these raw spatial coordinates into physical movement recognition, the system calculates the interior angles formed by intersecting biological joints. By tracking how these angles change over time against a defined baseline, the system can deterministically track human motion. Implementation Logic (Example: Bicep Curl): Point Identification: The algorithm isolates the spatial coordinates for three specific physiological landmarks: Point A (x1, y1): The Shoulder joint (acts as the anchor). Point B (x2, y2): The Elbow joint (the vertex of the angle). Point C (x3, y3): The Wrist joint (acts as the moving lever). Vector Calculation: The system calculates the inverse tangent (arctan2) to find the angle of the two vectors connecting the joints in 2D space. The arctan2 function is utilized instead of standard arctan because it correctly handles the varying quadrants of the Cartesian coordinate system, preventing division-by-zero errors when the arm is perfectly vertical. Angle Formula: The interior angle at the elbow (Point B) is calculated using the following trigonometric function, and then converted from radians to degrees: Angle_Radian = AbsoluteValue( arctan2(y3 - y2, x3 - x2) - arctan2(y1 - y2, x1 - x2) ) Angle_Degree = Angle_Radian * (180 / PI) Normalization: Because mathematical angles can exceed 360 degrees or return negative values, and biological joints do not naturally bend backward, the algorithm normalizes the result: If the Angle_Degree > 180, then Angle_Degree = 360 - Angle_Degree. State Machine (Repetition Counting): The calculated angle is continuously evaluated against predefined thresholds to prevent false positives. If Angle_Degree > 160 degrees, the arm is considered "extended" (State = Down). If Angle_Degree < 30 degrees, the arm is considered "flexed" (State = Up). When the state transitions sequentially from "Down" to "Up" and back to "Down", the algorithm increments the repetition counter by 1. Key Algorithmic Advantages: Client-Side Execution: The mathematical processing occurs entirely within the user's browser, eliminating the need to transmit video feeds to the backend and preserving strict user privacy. Device Agnosticism: Because the algorithm relies on relative angles rather than absolute pixel distances, it functions accurately regardless of the user's distance from the camera or the device's screen size. 7.5 Code Snippets (Core Implementation) This section highlights the critical logic responsible for the intelligence, tracking, and security of the Elevate platform.
