Vite was selected over traditional bundlers because it provides fast Hot Module Replacement (HMR) and optimized production compilation. Tailwind CSS: A utility-first CSS framework used to style the React components. It ensures that the application is fully responsive and automatically adapts to varying screen sizes, from mobile devices to desktop monitors. React Router DOM: Implemented to handle client-side routing, enabling smooth navigation between the login screen, profile settings, and the live workout environment without requiring full page reloads. Axios: Used for executing asynchronous HTTP requests to facilitate communication between the React frontend and the backend REST APIs. 7.1.2 Computer Vision and Kinematics Google MediaPipe (JavaScript API): A critical technology for the real-time pose tracking feature. MediaPipe is integrated directly into the React frontend. It captures the user's webcam feed and utilizes WebAssembly to process the video frames locally in the browser, extracting 33 distinct 3D spatial landmarks representing the human body. Because it runs client-side, it eliminates the latency of uploading video to a server. 7.1.3 Primary Backend (Authentication and Routing) The primary backend acts as a gateway, securely handling standard web operations. Node.js & Express.js: Node.js provides the runtime environment, while Express.js serves as the web framework. This server is responsible for handling user authentication routes, retrieving profile statistics, and logging completed workout sessions to the database. JSON Web Token (JWT): Implemented to securely transmit information between the React client and the Node.js server. JWT provides stateless session management and secure user authentication across requests. bcrypt: A cryptographic library utilized to securely hash user passwords before storing them in the database, ensuring that raw credentials are never stored in plain text. 7.1.4 Artificial Intelligence Backend (Inference Server) A separate server dedicated to executing machine learning and data processing tasks. Python & FastAPI: Python provides the foundation for the mathematical operations, while FastAPI provides the asynchronous framework to wrap the machine learning models into accessible REST APIs. Uvicorn: Used as the ASGI web server for hosting the FastAPI application and handling incoming asynchronous API requests on a dedicated port. XGBoost: The core machine learning library. Extreme Gradient Boosting (XGBoost) regression algorithms are implemented to predict the optimal workout sets, repetitions, and daily macronutrient targets based on the user's physical profile. Scikit-Learn & Pandas: Utilized to preprocess the raw data arrays, encode categorical variables, and structure the dataframes before feeding them into the XGBoost models for inference. Joblib: Used to load pre-trained machine learning (.pkl) models into memory at runtime. 7.1.5 Database and Storage MongoDB Atlas: A cloud-hosted NoSQL document database. MongoDB was selected because its document-oriented structure provides flexibility for storing user profiles, workout plans, nutritional information, and historical activity records. Mongoose: An Object Data Modeling (ODM) library deployed on the Node.js server to enforce strict validation rules and schemas on the data before it is inserted into the MongoDB cluster. 7.2 Development Environment The development environment for Elevate was configured to support the simultaneous execution of three independent systems: the user interface, the primary authentication backend, and the artificial intelligence server. Establishing a consistent, modular development environment was critical to ensure that all subsystems could communicate securely during the implementation phase. 7.2.1 Workspace Architecture The project utilizes a monorepo architecture, ensuring that all frontend code, backend routing logic, and machine learning models are version-controlled within a single repository. The workspace is logically partitioned into three distinct environments: Frontend Environment: Contains the Vite-compiled React source code, routing logic, and Tailwind configuration. Primary Backend Environment: Contains the Express.js routing logic, Mongoose database models, and authentication controllers. Machine Learning Environment: Contains the XGBoost training scripts, .pkl model files, and the FastAPI inference server. 7.2.2 Package Management and Dependency Isolation To prevent version conflicts during development, strict package management protocols were enforced. Node Package Manager (npm): Utilized to manage JavaScript dependencies for both the frontend and primary backend environments, ensuring libraries such as React, Axios, and Mongoose remain up to date. Python Virtual Environments (.venv): A dedicated virtual environment was established for the machine learning logic. This isolated the Python dependencies (such as XGBoost, FastAPI, and Pandas) from the global operating system, ensuring the inference environment remained reproducible across different workstations. pip was used to manage these dependencies via a strict requirements configuration. 7.2.3 Environment Configuration and Security Sensitive configuration data required to run the local development servers was strictly excluded from version control to prevent accidental exposure. Environment Variable Files (.env): Configuration files were deployed locally in both backend environments. These files securely housed the MongoDB Atlas connection string (URI), the JSON Web Token (JWT) cryptographic secret key, and the specific localhost port assignments necessary for Cross-Origin Resource Sharing (CORS) configuration. 7.2.4 Local Server Execution During development and testing, all three services were executed concurrently in separate runtime environments to simulate a full-stack cloud architecture. Frontend Execution: The React application was launched utilizing Vite's development server, binding to a local port (typically Port 5173). Vite’s Hot Module Replacement (HMR) allowed the user interface to update instantly upon saving code changes. Node.js Execution: The Express server was executed utilizing Nodemon, binding to a standard API port (typically Port 5000). Nodemon automatically restarted the primary backend server whenever authentication or database routing logic was modified. FastAPI Execution: The Python inference server was executed using FastAPI and Uvicorn, binding the application to a dedicated inference port (typically Port 8000) with auto-reload capabilities enabled. 7.2.5 Version Control and Collaboration Git was utilized as the primary version control system throughout the development lifecycle.
