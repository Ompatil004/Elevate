This strategy encompasses the following theoretical methodologies and their project-specific implementations: 1. Unit Testing Unit testing is the lowest level of software testing, where individual units or components of a software are tested in total isolation. A "unit" is the smallest testable part of an application, such as a specific function, method, or class. Unit testing is primarily a white-box testing technique performed by developers to ensure that internal algorithms operate correctly according to their design specifications. By isolating a section of code and verifying its correctness, unit tests facilitate early bug detection, simplify debugging, and ensure that localized code changes do not inadvertently introduce regressions. Application to Elevate: In the Elevate project, unit testing was critical for validating mathematical and algorithmic integrity. Computer Vision (Frontend): The mathematical logic underpinning the computer vision engine was rigorously unit-tested. Specifically, the calcAngle function within PoseDetector.jsx was isolated to verify that its arctan2 trigonometric formulas accurately computed interior joint angles. The state machine responsible for repetition counting was similarly tested to ensure that the defined Range of Motion (ROM) thresholds and hysteresis bands correctly prevented false positives. Machine Learning Pipeline (Backend): The Python feature derivation pipeline was tested to guarantee determinism. Functions like calculate_derived_features() in feature_pipeline.py were supplied with mock physiological data to verify the mathematical accuracy of the derived Body Mass Index (BMI), recovery scores, and progressive overload deltas before these values were fed into the XGBoost algorithm. Security Authentication: The Node.js auth.js middleware underwent unit testing to confirm the correct cryptographic decryption and validation of JSON Web Tokens (JWT), ensuring that expired or tampered tokens were automatically rejected. 2. Integration Testing Integration testing follows unit testing and focuses on combining individual units and testing them as a group. The primary objective is to expose faults in the interaction between integrated units. In distributed and service-oriented architectures, integration testing is crucial for validating API contracts, data serialization, and communication protocols across different application layers. It ensures that data structures generated by one module are correctly interpreted by another, mitigating the risk of structural mismatches. Application to Elevate: Because Elevate employs a polyglot monorepo architecture (React, Node.js, and Python), integration testing was prioritized across the REST API boundaries. Cross-Service Communication: Tests verified that the Node.js API gateway successfully routed authenticated user requests to the Python FastAPI inference engine via the /generate-plan endpoint. It was essential to prove that the JSON payloads transmitted by Node.js strictly adhered to the schema expected by the Python backend. Database Serialization: Integration tests validated the Data Access Object (DAO) layer. It ensured that the deeply nested, hierarchical machine learning workout plans generated by the Python XGBoost engine were correctly serialized and stored into the MongoDB database using Mongoose object mapping schemas without data loss. 3. System Testing (End-to-End) System testing is a level of testing that validates the complete and fully integrated software product. Unlike unit and integration testing, system testing falls under the umbrella of black-box testing, meaning it evaluates the system's compliance with its specified functional and non-functional requirements without knowledge of the internal code structure. It simulates real-world environments to ensure that all subsystems work together harmoniously to fulfill the overarching user objectives. Application to Elevate: System testing for Elevate evaluated the entire application lifecycle from the perspective of an end user (End-to-End or E2E testing). User Journey Validation: The complete user flow was simulated, starting from account registration and login, to the completion of the biometric assessment questionnaire. The system test verified that the data successfully traveled to the machine learning engine, returning a personalized workout plan to the React UI. Real-Time Interaction: Further system tests validated the live workout interface. This included launching the camera, detecting human poses, executing repetitions, and verifying that form correction feedback was delivered to the user in real-time, concluding with the successful saving of the session data to the backend. 4. Performance and Non-Functional Testing While functional testing verifies "what" the system does, non-functional testing evaluates "how well" it does it. This encompasses performance, load, stress, and latency testing. Performance testing is essential to ensure that the system meets strict quality-of-service metrics, such as response times, resource utilization, and throughput, particularly under computationally heavy operations. Application to Elevate: Given Elevate’s reliance on intensive real-time AI and machine learning tasks, non-functional testing was critical. Client-Side Evaluation: The real-time computer vision engine relies heavily on client-side browser capabilities. Performance testing verified that the MediaPipe pose estimation model executed efficiently without blocking the main JavaScript thread. The computer vision module was evaluated to ensure smooth real-time pose tracking on standard consumer hardware without noticeable lag or excessive memory consumption. Inference Latency: The Python FastAPI backend was tested to ensure the Joblib module could rapidly load the pre-trained XGBoost models into memory.
