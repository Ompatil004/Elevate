Passwords are securely hashed using the bcrypt algorithm before being stored in the MongoDB database. 5.4.5 System Architecture Planning The deployment architecture is designed using a modular service-oriented architecture to prevent heavy AI computations from impacting the primary web server. The system is structurally planned around a classic three-tier deployment model: 1. Client Tier (Presentation Layer): The React frontend operates as a decoupled Single Page Application (SPA). This layer is responsible for rendering the user dashboard, managing state, and executing the client-side Google MediaPipe vision models directly within the user's browser. 2. Application Tier (Logic Layer): To ensure stable operation, the backend logic is split across two distinct environments: Node.js Server: Handles standard administrative tasks, user authentication, routing, and basic database queries. Python FastAPI Server: Acts as a dedicated AI processing server entirely focused on running XGBoost machine learning inferences for the workout and nutrition engines. 3. Data Tier (Storage Layer): The database layer utilizes MongoDB Atlas to provide a secure, scalable NoSQL document storage solution. This layer securely houses all user profiles, encrypted credentials, and historical session logs. This decoupled topology ensures that the frontend communicates efficiently with both the Node.js server for data retrieval and the FastAPI server for AI generation, leading to improved responsiveness and maintainability.
